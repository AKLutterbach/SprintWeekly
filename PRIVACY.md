# Privacy Policy for Sprint Weekly

**Last Updated:** December 2, 2025

## Overview
Sprint Weekly is a Jira app that generates sprint reports. This privacy policy explains how we handle data.

## Data Collection
Sprint Weekly accesses the following data from your Jira instance:
- Project information (names, keys)
- Sprint information (names, dates, states)
- Issue data (summaries, statuses, types, priorities, assignees)
- Issue metadata (created dates, resolution dates, story points)

## Data Storage
- **Temporary Caching:** Report data is cached for up to 30 minutes to improve performance
- **No Long-term Storage:** We do not store your Jira data beyond the temporary cache
- **No Third-party Sharing:** Your data is never shared with third parties
- **Atlassian Infrastructure:** All data is processed within Atlassian's Forge platform

## Data Usage
Data is used solely to:
- Generate sprint reports and metrics
- Display issue summaries and statistics
- Export reports to PDF format

## Data Security
- All API requests use Atlassian's secure authentication (`asUser()`)
- Data transmission occurs within Atlassian's infrastructure
- Cached data automatically expires after 30 minutes

## User Rights
You can:
- Request data deletion by uninstalling the app
- Contact us at [your-email@example.com] for privacy questions

## App Permissions
Sprint Weekly requires the following Jira permissions:
- `read:jira-work` - Read Jira project and issue data
- `read:issue:jira` - Read issue details
- `read:project:jira` - Read project information
- `read:issue-details:jira` - Read detailed issue information
